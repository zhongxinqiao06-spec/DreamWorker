## Instructions

Create a PRD draft that includes:

- Problem statement and target users.
- Jobs-to-be-done and primary scenarios.
- MVP scope and explicit out-of-scope items.
- Core user flows and states.
- Functional requirements.
- Acceptance criteria.
- Open questions and validation risks.

Prefer sharp tradeoffs over broad feature lists.
