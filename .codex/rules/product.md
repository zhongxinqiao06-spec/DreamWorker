# Product Rules

DreamWorker 的产品判断以 idea-to-launch 的垂直场景为中心。

## 定位

- DreamWorker 是开放式 Agent 项目孵化台，不是通用工作流工具。
- 首版服务独立开发者、AI 创业者和小型产品团队。
- 核心价值是把想法转成可信分析、可执行蓝图和可交付 artifact。

## 聚焦

- 优先完成 idea -> Dream Brief -> Research Pack -> Blueprint -> Artifact 的闭环。
- 不为了“多 Agent”而多 Agent；少 Agent、高质量、可解释优先。
- 不追求第一版连接所有平台；先把 capability 抽象、审批和审计做好。
- 不做通用个人 Agent OS，避免和已有通用产品正面重叠。

## 差异化

每个重要输出都应体现：

- Evidence-first。
- Blueprint-first。
- Open Agent Bus。
- Human-in-command。
- Idea-to-launch。

## 输出质量

- 结论必须给出证据、置信度、假设和待验证实验。
- Blueprint 必须能转成任务和交付物。
- 用户必须能看懂当前系统为什么这么规划、下一步要做什么、风险在哪里。
