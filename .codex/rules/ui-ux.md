# UI/UX Rules

DreamWorker 是工作台，不是营销落地页。首屏应该是可用的项目操作界面。

## 核心视图

- Idea Chat：接收想法、追问、解释规划、接受用户 steering。
- Blueprint Canvas：展示任务蓝图、依赖、Agent 分工、审批点和可交付物。
- Run Console：显示实时事件、工具调用、任务状态、失败、成本和证据。
- Capability Panel：管理内置能力、MCP、Skill、API、Agent 和人工任务。
- Artifact Hub：查看、编辑、导出 PRD、roadmap、issues、文案、schema。
- Approval Center：集中处理需要用户确认的动作。

## 体验原则

- 用户始终知道系统正在做什么、为什么做、下一步是什么。
- 高风险动作必须先预览再确认。
- 证据、成本、风险和失败原因必须可见。
- Blueprint 可编辑，Run 可暂停，任务可重排。
- 空状态要引导用户输入 idea，而不是展示宣传文案。

## 界面风格

- 面向生产力工作台，信息密度适中，视觉克制。
- 优先使用清晰分栏、列表、时间线、状态标签和工具卡片。
- 不用大面积装饰性渐变、营销式 hero 或无关插画。
- 按钮、菜单、开关、滑块、标签页等控件使用明确语义。
- 文案以中文为主，关键协议名和字段名保留英文。
